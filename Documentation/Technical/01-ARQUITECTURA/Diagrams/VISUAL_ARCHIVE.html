Estructura de Datos (ERD)

Propósito: Definir la persistencia y relaciones críticas del e-commerce.
Riesgo: Sin este mapa, la integridad referencial en pedidos y stock es vulnerable.
erDiagram profiles ||--o{ orders : "coloca" products ||--o{ order_items : "incluido_en" orders ||--|{ order_items : "contiene" profiles { uuid id PK string email UK string full_name string role } products { bigint id PK string category jsonb name numeric price integer stock } orders { uuid id PK uuid user_id FK string status numeric total_amount jsonb shipping_address } order_items { bigint id PK uuid order_id FK bigint product_id FK integer quantity numeric price_at_time }

🔍 Análisis de Nodos

Polimorfismo i18n Los campos Name/Description usan JSONB permitir multilingüismo (ES/EN) sin duplicidad de registros.
Integridad de Precios `price_at_time` en order_items congela el valor al momento de la compra para auditorías legales.
