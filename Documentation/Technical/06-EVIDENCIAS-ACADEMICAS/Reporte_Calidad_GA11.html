Informe de Buenas Prácticas de Calidad de Software

Aplicación documentada en las disciplinas de calidad

Resumen del Proyecto:
Este documento detalla la integración de normas ISO/IEC 25000 y prácticas de Clean Code en el desarrollo de la plataforma Origen Sierra Nevada.

1. Introducción

El presente documento detalla la aplicación de buenas prácticas de calidad en el desarrollo del proyecto de software Origen Sierra Nevada, una plataforma de comercio electrónico especializada en café de origen. El objetivo es evidenciar cómo se han integrado estándares y metodologías para asegurar un producto robusto, mantenible y escalable.

2. Modelo de Calidad (ISO/IEC 25000)

Se ha seleccionado el modelo SQuaRE, priorizando características críticas para el e-commerce:

  • Mantenibilidad: Capacidad del código para aceptar cambios futuros sin introducir errores.
  • Adecuación Funcional: Alineación total con los requisitos de venta y gestión de usuarios.
  • Fiabilidad: Robustez ante fallos y manejo de excepciones en tiempo real.

3. Implementación Técnica

3.1. Tipado Estático (TypeScript)

El uso de TypeScript reduce drásticamente los errores en tiempo de ejecución y sirve como documentación viva del código.

Archivo: types.ts
export interface Product {
    id: string;
    category: ProductCategory; 
    name: Multilingual; // Soporte i18n nativo
    price: number;
    intrinsics?: ProductPersonality; // Propiedades extendidas
}
export type UserRole = 'admin' | 'user' | 'colaborador';

3.2. Arquitectura Modular

Siguiendo principios SOLID, se ha separado la lógica (Servicios) de la vista (Componentes) y el estado global (Contextos).

  • /components: Elementos UI atómicos y reutilizables.
  • /services: Capa de abstracción para comunicación con Supabase.
  • /contexts: Manejo de estado seguro (Auth, Carrito).

3.3. Clean Code & Nomenclatura

Se establecieron convenciones estrictas para mantener la consistencia:

  • PascalCase: Componentes (e.g., ProductCard.tsx).
  • camelCase: Funciones y hooks (e.g., useCart).
  • Variables Semánticas: Nombres descriptivos que revelan la intención (e.g., isUserAuthenticated en lugar de flag).

3.4. Gestión de Estado y Seguridad

La autenticación se centraliza en un Contexto de React que expone el estado del usuario a toda la aplicación, simplificando la protección de rutas.

Interface AuthState
export interface AuthState {
    user: User | null;
    profile: Profile | null;
    loading: boolean;
    isAdmin: boolean; // Flag directo para control de acceso
}

4. Herramientas QA

El stack tecnológico incluye herramientas de aseguramiento continuo:

  • Vite: Empaquetado rápido y optimizado para producción.
  • ESLint: Análisis estático para detectar patrones de error.
  • Git: Control de versiones con estrategia de ramas por funcionalidad.

5. Conclusiones

La implementación de estas buenas prácticas en Origen Sierra Nevada ha resultado en un código autodocumentado, una arquitectura escalable y una reducción significativa de la deuda técnica inicial, asegurando que el producto final cumpla con los estándares de calidad industrial.
