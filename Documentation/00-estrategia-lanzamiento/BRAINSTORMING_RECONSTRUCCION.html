Reorganizar componentes en carpetas por feature: /landing, /admin, /client, /payments, /finance. Estado global con Zustand o Context API.

A4

Historial de pedidos, método de pago favorito, productos frecuentes y estado del pedido actual con línea de tiempo visual.

QR Dinámico por Compra

WhatsApp Integrado por Producto
Botón flotante persistente + wa.me con mensaje pre-llenado por producto. Flujo: consulta → precio → pago → confirmación.
A7
Newsletter Automático con Resend
Resend ya está instalado. Flujo: registro → welcome email → alertas de nuevos productos → promociones segmentadas.
A8
Sistema Financiero — Charts de Proyección
Ingresos vs egresos, flujo de caja mensual, proyección a 3/6/12 meses. Charts interactivos con Recharts o Chart.js.

Arquitectura React Modular por Feature

Es la base que hace posible el resto. Sin una arquitectura limpia, cada nueva feature (pagos, finanzas, IA) se convierte en deuda técnica. Organizar por feature-folders permite que cada módulo escale independientemente y que el equipo trabaje en paralelo sin conflictos.

Impacto: 5/5 Viabilidad: 5/5 Claridad: 5/5
Primer Paso Crear estructura de carpetas: /features/{landing,admin,client,payments,finance,auth} + configurar Zustand para estado global.
React 19 TypeScript Zustand React Router v7
2
✨

Hero Carrusel Inmersivo Glassmorphism

La primera impresión determina si el visitante compra o se va. Un hero carrusel con parallax de la despensa — con glass cards flotantes mostrando producto, origen y precio — comunica premium instantáneamente. Es la principal herramienta de conversión tanto para compradores como para proveedores.

Impacto: 4/5 Novedad: 4/5 Conversión: ★★★★★
Primer Paso Crear componente HeroCarousel.tsx con Swiper.js o Embla + CSS backdrop-filter: blur(12px) sobre imágenes existentes de la despensa.
#141E16 / #C8AA6E Playfair Display CSS Glassmorphism Parallax
3
🔐

Ciberseguridad: JWT + Supabase RLS + Auditoría

Con pagos reales y datos financieros en juego, la seguridad no es opcional. Supabase RLS garantiza que cada rol (admin/cliente/proveedor) solo acceda a sus datos. JWT con rotación automática + registro de auditoría protege sin agregar fricción al usuario.

Impacto: 5/5 Viabilidad: 5/5 Seguridad: ★★★★★
Primer Paso Habilitar RLS en todas las tablas de Supabase + crear políticas por rol + tabla de auditoría con triggers para acciones sensibles.
Supabase RLS JWT Bcrypt Audit Log
4
💰

Zona de Pagos: QR Dinámico + Billeteras + WhatsApp

Unificar todos los métodos de pago colombianos (Nequi, Daviplata, PSE, llave/alias) en un solo flujo reduce el abandono de compra. El QR se genera con el monto exacto del pedido y el WhatsApp actúa como canal de confirmación y soporte — sin necesidad de API de pago costosa para el MVP.

Impacto: 4/5 Viabilidad: 5/5 Sin garantías: ✓
Primer Paso Crear componente PaymentZone.tsx con QR generator (qrcode.react) + lista de métodos de billetera + botón de confirmación vía WhatsApp con referencia de pago.
qrcode.react wa.me API Nequi / Daviplata PSE / Llave
5
📊

Control Financiero + Big Data + Proyección

El sistema financiero convierte origen_sierranevada de tienda en negocio inteligente. GA4 para Big Data desde día 1, Supabase para datos transaccionales, y charts de proyección con Recharts que muestran flujo de caja, inventario crítico y crecimiento esperado. El admin ve el negocio completo en una pantalla.

Impacto: 5/5 Big Data: ✓ Proyección: 3/6/12 meses
Primer Paso Instalar Recharts + crear FinanceDashboard.tsx con 4 cards (ventas hoy, inventario, flujo mensual, proyección) + conectar a tablas de Supabase.
Recharts Supabase Realtime Google Analytics 4 jsPDF

Módulos Legales Agregados — Factura Electrónica + Compliance Colombia

Dos módulos críticos incorporados al alcance de la reconstrucción, necesarios para operar legalmente en Colombia como comercio electrónico con pagos reales.

🧾 Factura Electrónica DIAN
• Resolución 000042/2020 — obligatoriedad
• Formato UBL 2.1 (XML con firma digital)
• Integración vía PTA autorizado (Alegra/Siigo API)
• PDF descargable con cufe + QR de validación
• Numeración autorizada por rango DIAN
• Aplicable: ventas > $0 o persona jurídica
⚖️ Compliance Legal Colombia
• Ley 1581/2012 — Protección de datos (SIC)
• Política de Privacidad + aviso visible en landing
• Consentimiento explícito en formularios
• Resolución 1519/2020 — WCAG 2.1 accesibilidad
• Términos y Condiciones del comercio
• Régimen SIMPLE o IVA según clasificación DIAN
