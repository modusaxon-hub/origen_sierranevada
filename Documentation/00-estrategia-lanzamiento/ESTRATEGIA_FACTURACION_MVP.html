⚙️ Implementación Técnica (F3-005)

Cambio Principal en Backend

El cálculo de IVA DEBE ser 0 en todas las llamadas a la API del PTA (Alegra/Siigo). No se trata de "omitir" el campo, sino de enviarlo explícitamente como 0.

1️⃣ Cálculo en Checkout
// CORRECTO para NO Contribuyente const calculateTotal = (items) => { const subtotal = items.reduce((sum, item) => sum + (item.price * item.quantity), 0); const iva = 0; // SIEMPRE 0 const total = subtotal + iva; return { subtotal, iva, total }; }; // Ejemplo: // Café Malú $18.000 x 1 // Subtotal: $18.000 // IVA: $0 // Total: $18.000
2️⃣ Payload a PTA (Alegra API)
{ "invoice": { "seller": { "nit": "XXXXXXXXX-X", "name": "Café Origen Sierra Nevada E.U." }, "buyer": { "id": "CCXXXXXXXX", // Recolectado en checkout "name": "Cliente" }, "items": [{ "description": "Café Malú x1", "quantity": 1, "unit_price": 18000, "subtotal": 18000, "tax_percentage": 0, // ← IMPORTANTE: 0, no null "tax_amount": 0 }], "subtotal": 18000, "total_tax": 0, // ← IVA Total = 0 "total": 18000, "note": "No responsable de IVA - Art. 437 ET" } }
